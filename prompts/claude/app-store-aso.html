📋 Prompt
You are an App Store Optimisation specialist who has improved app download rates by 200%+.

App name: [name]
Category: [productivity/gaming/health/utility/social]
Core function: [what the app does in one sentence]
Target user: [who it's for]
Competitors: [2-3 competitor apps]
Advantage: [what makes it better or different]

Task:

iOS APP STORE:
1. TITLE (30 chars max): App name + highest-volume keyword
2. SUBTITLE (30 chars max): Secondary benefit + keyword
3. KEYWORDS FIELD (100 chars): Comma-separated, no spaces around commas, no repeats from title/subtitle
4. DESCRIPTION (4,000 chars): First 3 lines = conversion hook (shown before 'more'); Features as benefits; Social proof; CTA

GOOGLE PLAY:
5. TITLE (50 chars): More keyword-rich than iOS title
6. SHORT DESCRIPTION (80 chars): Most important benefit in one sentence
7. FULL DESCRIPTION (4,000 chars): Keyword-rich, feature list, social proof, CTA

SCREENSHOT COPY:
8. 5 captions (under 30 chars each) in priority order

ASO NOTES: Top 3 keyword opportunities + 1 localisation tip

💡 Pro Tips
First 3 description lines are the only ones most users read — treat them as your entire pitch
App title is the most important ASO factor — keyword in title carries 3× more weight than anywhere else
Screenshots convert browsers to installers — first screenshot shows core feature, not a welcome screen
Localising to key languages can double downloads at minimal cost
⚠️ Common Mistakes
App title that's just the app name — wasted opportunity for the most important ASO element
Description reads like a press release — benefit-focused and conversion-optimised, not corporate
Repeating title keywords in keywords field — Apple already knows; you're wasting character slots
No social proof — 'used by 50,000 professionals' significantly changes conversion rates
❓ FAQ
  • How long for ASO to work?
    iOS: ranking changes within 1-2 weeks post-update. Google Play: sometimes within days. Consistent optimisation over 3-6 months produces significantly better cumulative results.
  • Same description on iOS and Google Play?
    No. iOS: keyword field handles SEO; description should be human-first. Google Play: description is indexed by Google — integrate keywords naturally.
  • How to research ASO keywords?
    Free: App Store search autocomplete, competitor title/subtitle analysis. Paid: Sensor Tower, AppTweak. Also Google Trends for category-level terms.
  • Best model?
    ChatGPT for app store listings — concise benefit-forward style matches the format. Claude for longer description copy where trust-building matters.
