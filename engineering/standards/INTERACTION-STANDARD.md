# INTERACTION STANDARD

## Purpose

A calculator should feel interactive rather than static.

Educational content should respond to user actions whenever practical.

---

## Result Experience

After calculation the page should:

- Highlight the result.
- Allow easy copying.
- Encourage the next useful action.
- Keep the user engaged.

## Empty State

Before calculation:

Provide helpful guidance.

---

## Success State

After calculation:

Show confidence.

Highlight the answer.

Provide useful follow-up actions.

---

## Error State

Explain errors clearly.

Never blame the user.

Explain how to fix invalid input.
