# CONTENT Rules

## Purpose

Ensure every page provides accurate, useful, trustworthy, and complete information that helps users accomplish their goal.

Content exists to solve problems, not to fill space.

---

# Core Principles

Every page must:

- Answer the user's primary question.
- Explain concepts clearly.
- Prioritize accuracy over volume.
- Provide practical value.
- Remain easy to scan.

Never write content only to increase page length.

---

# User First

Every section must answer one question:

**Why does this help the user?**

If no clear answer exists,

remove the section.

---

# Accuracy

All information must be:

✓ Factually correct

✓ Current

✓ Verifiable

✓ Consistent

Do not guess.

Do not invent facts.

---

# Clarity

Write for humans first.

Use:

- Short paragraphs
- Clear headings
- Simple language
- Logical structure

Avoid unnecessary jargon.

When technical terms are required,

explain them.

---

# Educational Value

Every page should help users understand:

- What it is
- Why it matters
- How it works
- When to use it
- Common mistakes

Teaching builds trust.

---

# Examples

Whenever appropriate,

include practical examples.

Examples should:

✓ Reflect realistic situations

✓ Be easy to follow

✓ Match calculator behavior

Avoid artificial examples.

---

# Formulas

When calculations are performed,

include:

- Formula
- Variable definitions
- Units
- Assumptions

Users should understand how results are produced.

---

# Tables

Use tables only when they improve understanding.

Examples:

- Conversion tables
- Comparison tables
- Reference tables

Do not create decorative tables.

---

# FAQs

FAQs should answer genuine user questions.

Rules:

✓ Based on user intent

✓ Short

✓ Accurate

✓ Non-repetitive

Do not create filler questions.

---

# References

Whenever possible,

support factual statements using authoritative sources.

Examples:

- Government agencies
- International standards
- Official documentation
- Academic publications

Avoid unsupported claims.

---

# Consistency

Terminology must remain consistent throughout the page.

Do not alternate between different names for the same concept.

---

# Readability

Every page should be easy to scan.

Prefer:

- Bullet lists
- Tables
- Examples
- Short sections

Avoid large blocks of text.

---

# Completeness

A page is complete when users can:

✓ Understand the topic

✓ Use the calculator correctly

✓ Interpret the results

✓ Learn related concepts

Without needing another website.

---

# Content Quality Checklist

Every page should include:

✓ Clear introduction

✓ Accurate explanations

✓ Practical examples

✓ Formula (if applicable)

✓ FAQs

✓ References (when applicable)

✓ Related tools

---

# Completion

Content Rules are satisfied when the page is:

- Helpful
- Accurate
- Understandable
- Complete
- Trustworthy
