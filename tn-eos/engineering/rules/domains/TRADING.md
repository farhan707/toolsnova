# Trading Domain

## Purpose

Ensure trading tools promote informed decision-making through accurate calculations and transparent methodology.

Trading tools support analysis.

They do not predict markets.

---

# Core Principles

Every trading calculator must:

- Produce deterministic calculations.
- Explain assumptions.
- Define every variable.
- Avoid predictive language.
- Encourage responsible risk management.

---

# Trading Concepts

Explain terminology whenever used.

Examples:

- Margin
- Leverage
- Pip
- Lot Size
- Drawdown
- Risk Reward
- Swap
- Spread
- Equity
- Free Margin

Never assume users understand trading jargon.

---

# Risk Management

Whenever applicable explain:

✓ Position sizing

✓ Capital preservation

✓ Risk percentage

✓ Leverage implications

✓ Margin requirements

Trading tools should educate, not encourage excessive risk.

---

# Formula Transparency

Every calculation should include:

✓ Formula

✓ Variables

✓ Units

✓ Assumptions

✓ Interpretation

---

# Practical Examples

Use realistic market scenarios.

Avoid unrealistic account balances or leverage.

Examples should match calculator behavior.

---

# Educational Value

Users should understand:

- What is being calculated

- Why it matters

- How to use the result

- Common mistakes

---

# Neutrality

Never imply:

- Guaranteed profit

- Winning strategies

- Market predictions

Maintain educational neutrality.

---

# References

Prefer official sources.

Examples:

- Broker documentation

- Exchange specifications

- Regulatory publications

- Platform documentation

---

# Completion

Trading requirements are satisfied when users can:

✓ Understand the calculation

✓ Apply the result correctly

✓ Recognize trading risks

✓ Interpret assumptions
