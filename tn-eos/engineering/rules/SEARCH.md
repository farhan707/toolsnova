# SEARCH Rules

## Purpose

Ensure every page is easy to discover, understand, navigate, and trust by both humans and AI-powered search systems.

Search quality is achieved by building pages that deserve to be discovered—not by manipulating search engines.

---

# Core Principles

Every page must:

- Solve a real user problem.
- Clearly communicate its purpose.
- Be understandable without external context.
- Demonstrate topical authority.
- Connect naturally to related knowledge.

Never optimize for search engines at the expense of users.

---

# Search Intent

Before implementation, identify the dominant search intent.

Supported intents:

- Informational
- Transactional
- Navigational
- Commercial Investigation

Every page should primarily satisfy one intent.

Do not mix unrelated intents.

---

# Primary Entity

Every page must define one primary entity.

Examples:

- Height Converter
- Margin Calculator
- JSON Formatter
- EMI Calculator

The primary entity should appear naturally in:

- Title
- Main heading
- Introduction
- Structured data
- Internal links

---

# Secondary Entities

Support the primary entity using closely related concepts.

Examples:

Height Converter

- Feet
- Inches
- Centimeters
- Meters
- Metric System
- Imperial System

Avoid unrelated keywords.

---

# Metadata

Every page requires:

✓ Unique title

✓ Unique meta description

✓ Canonical URL

✓ Open Graph metadata

✓ Twitter metadata

Metadata must accurately describe the page.

Never use clickbait.

---

# Structured Data

Use structured data whenever applicable.

Supported schemas include:

- WebPage
- SoftwareApplication
- FAQPage
- BreadcrumbList

Structured data must match visible page content.

Never create hidden or misleading schema.

---

# Internal Linking

Every page should connect naturally to related pages.

Rules:

✓ Link only when helpful.

✓ Use descriptive anchor text.

✓ Prioritize closely related tools.

✓ Avoid excessive linking.

---

# Content Discoverability

Content should be organized logically.

Preferred order:

1. Purpose
2. Tool
3. Explanation
4. Examples
5. FAQ
6. References
7. Related Tools

Users should quickly locate the information they need.

---

# Entity Coverage

Cover the topic completely.

Explain:

- Definitions
- Relationships
- Common terminology
- Practical usage
- Common mistakes

Avoid shallow coverage.

---

# AI Search Readiness

Pages should be understandable by AI systems.

Content should:

✓ Be factual

✓ Be well structured

✓ Use consistent terminology

✓ Explain concepts clearly

✓ Avoid ambiguity

---

# Trust Signals

Whenever appropriate include:

- References
- Methodology
- Assumptions
- Last Updated
- Authoritative sources

Trust is earned through transparency.

---

# Completion Checklist

A page satisfies the Search Rules when:

✓ Primary entity is clearly defined

✓ Search intent is satisfied

✓ Metadata is complete

✓ Structured data is valid

✓ Internal links are useful

✓ Entity coverage is complete

✓ AI readability is high

✓ Trust signals are present
