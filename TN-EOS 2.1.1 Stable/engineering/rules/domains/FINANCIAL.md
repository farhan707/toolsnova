# Financial Domain

## Purpose

Ensure financial calculators are accurate, transparent, understandable, and trustworthy.

Financial calculations may influence real-world decisions.

Accuracy always takes priority over presentation.

---

# Core Principles

Every financial page must:

- Explain assumptions.
- Use deterministic calculations.
- Show calculation methodology.
- Clearly define variables.
- Avoid misleading claims.

---

# Precision

Display values using appropriate precision.

Currency

✓ Two decimal places unless justified.

Percentages

✓ Meaningful precision.

Ratios

✓ Consistent rounding.

---

# Formula Transparency

Whenever calculations exist, include:

✓ Formula

✓ Variables

✓ Units

✓ Assumptions

✓ Limitations

Users should understand how results are produced.

---

# Terminology

Define important financial concepts.

Examples:

- Interest
- ROI
- Margin
- Equity
- Leverage
- APR
- Profit
- Cost

Never assume prior knowledge.

---

# Practical Examples

Include realistic scenarios.

Examples should:

✓ Reflect real-world values

✓ Match calculator behavior

✓ Help interpretation

---

# Risk Communication

Where appropriate explain:

- Financial risk
- Assumptions
- Limitations

Remain educational.

Never provide investment advice.

---

# References

Prefer authoritative sources.

Examples:

- Government agencies
- Central banks
- International standards
- Official financial documentation

---

# Completion

Financial requirements are satisfied when users can:

✓ Understand the calculation

✓ Verify the methodology

✓ Interpret the result

✓ Recognize assumptions

✓ Identify the primary financial outcome quickly

---

# Result Interpretation

Financial calculators should present the most important decision-making information before detailed calculation output.

Primary results should be immediately understandable and support rapid interpretation without requiring users to read the complete breakdown.

The presentation method may vary according to the calculator's purpose, but should consistently prioritize:

✓ Primary financial outcomes

✓ Clear interpretation

✓ Decision support

Detailed calculations, assumptions, and supporting information should remain available for transparency and verification.

Domain-specific result metrics should reflect the calculator's intended purpose rather than using a fixed template.
